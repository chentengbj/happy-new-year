# 链接功能说明

## 问题说明

当前数据文件（`data.js`）中的 `source` 字段使用的是示例链接（`https://example.com/...`），这些链接无法实际访问。

## 解决方案

### 方案1：更新为真实链接（推荐）

编辑 `data.js` 文件，将示例链接替换为真实的新闻来源链接：

**修改前：**
```javascript
source: 'https://example.com/news1'
```

**修改后：**
```javascript
source: 'https://www.36kr.com/p/123456'  // 真实的新闻链接
```

### 方案2：使用数据收集脚本

运行数据收集脚本会自动获取真实的链接：

```bash
python3 collect_news.py
```

### 方案3：手动添加真实数据

在添加新数据时，确保 `source` 字段是有效的HTTP/HTTPS链接：

```javascript
{
    company: '好未来',
    title: '新闻标题',
    content: '新闻内容',
    priority: '重点',
    categories: ['AI教育'],
    date: '2025-10-02',
    source: 'https://真实新闻网站.com/article/123'  // 必须是真实可访问的链接
}
```

## 链接验证规则

系统会自动验证链接：

✅ **有效链接：**
- 以 `http://` 或 `https://` 开头
- 不是 `example.com` 域名
- 格式正确的URL

❌ **无效链接（会显示为禁用状态）：**
- `https://example.com/...`
- `http://example.com/...`
- 空链接或格式错误的链接

## 注意事项

1. **链接格式**：必须是完整的URL，以 `http://` 或 `https://` 开头
2. **链接有效性**：确保链接可以正常访问
3. **相对路径**：不支持相对路径，必须使用绝对路径
4. **链接安全性**：系统会自动添加 `rel="noopener noreferrer"` 提高安全性

## 测试链接

更新链接后，刷新浏览器页面，点击"查看原文"按钮：
- ✅ 如果链接有效：会在新标签页打开
- ❌ 如果链接无效：按钮会显示为灰色禁用状态

## 批量更新链接

如果需要批量更新所有示例链接，可以使用文本编辑器的查找替换功能：

1. 打开 `data.js` 文件
2. 查找：`https://example.com/`
3. 替换为：真实的新闻网站链接前缀
4. 逐个更新具体的文章链接
